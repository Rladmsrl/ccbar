问题

"橙色 working 跟绿色 attention 不好区分"

现在的两个色 (#FF9F1C 橙 & #40C864 绿) 在 saturated 端 色相分得开,但放到 36×~22pt 折叠条段里走 0.85→0.15 alpha 渐变后, 在 gradient 衰减端的亮度跟饱和度都拉平,只剩色相对比 — 而色相对比在 peripheral vision 里又不如亮度差敏感, 就糊到一起了。

另外两组潜在冲突也顺便审一下:
— thinking blue (200°) 跟 sweeping cyan (190°) 只差 10°,都是冷色
— working orange (32°) 跟 error red (3°) 都是暖色,fade 端容易混

设计目标

三层差异: 色相 · 亮度 · 饱和

每对状态色都应该靠 至少 2 层 拉开距离 (色相 + 亮度 / 色相 + 饱和 / 亮度 + 饱和),不能只靠 hue。小尺寸渐变下亮度差最稳。

语义约束保留:
— error 必须红 (universal)
— working 是暖色(active/energetic)
— thinking 是冷色(cerebral)
— attention 是"完成-看我"信号(可以走 cool 不走 grass green)
— juggling 是 multitask(紫色/品红族)
— sweeping 是 compaction(中性/灰蓝)

VARIANT №01 · minimal
Amber & Emerald
01

只动 2 个色:orange → 更黄的 #FFB020;green → 更深更冷的 翡翠绿 #10B981。这俩现在亮度差大 + 色相隔了 100°,fade 端也分得开。 其余 4 色不动,最低改造成本。

改了什么

  • working #FF9F1C → #FFB020(更黄/更亮,跟 error red 也拉开了)
  • attention #40C864 → #10B981(deep emerald,亮度低饱和高)

取舍

  • + 改动最小 — 4 个色不动,只调 2 个有冲突的
  • + 亮度差最大,fade 端依然分得开
  • − thinking 跟 sweeping 还是太靠近(留隐患)
VARIANT №02 · rebalance
Six Quadrants
02

6 色铺满色相环,每色相隔 60°,亮度也错开。orange 变更暖的 #FF7A1A, attention 走 turquoise #14C4A1,sweeping 退成纯灰蓝 #7B8AA0 (跟 thinking blue 区分开)。3 改动,体系最清爽。

改了什么

  • thinking #36A2EB → #3B82F6(royal blue,更深)
  • working → #FF7A1A(暖橙,跟绿色 hue 距离 100°+)
  • juggling → #C557E0(更亮的洋红)
  • attention → #14C4A1(turquoise,带 cyan 倾向)
  • sweeping → #7B8AA0(去饱和的灰蓝,跟 thinking 完全分开)
  • error → #EF4444(略深一点,纯红)

取舍

  • + 整套体系一次性 rebalance,所有相邻对都拉开
  • + sweeping 退成 muted gray-blue 反而更符合"compaction"语义
  • − attention 变 turquoise 后跟 thinking blue 还隔得开但是同冷色族
  • − 改动多,要更新所有 events 颜色映射 + a11y test
VARIANT №03 · editorial
Editorial
03

把整套 palette 从"霓虹 + 警告色"挪到"印刷品 + 杂志油墨"美学。 饱和度普遍降,但每色加 character。读起来不像 system UI 警告,更像 一支 fine pen 在条纹纸上写。一次性大改。

改了什么(全部)

  • thinking → #4F6FB8(粉笔蓝,深色)
  • working → #E8841F(烧橙/印刷橙)
  • juggling → #8B5CF6(薰衣草紫)
  • attention → #0D9488(深松石 teal)
  • sweeping → #A39584(纸张米色 — 不再是冷色)
  • error → #C53030(印章红,不再霓虹)

取舍

  • + 整体调性从"系统警告"挪到"杂志/印刷",更有品味
  • + 饱和度普遍降,长时间观看不刺眼
  • − error 失去"霓虹红"的警觉性,可能弱化紧急感
  • − sweeping 米色在深色背景上偏淡(可能弱到看不见)
  • − 跟 macOS 系统色脱节最大
