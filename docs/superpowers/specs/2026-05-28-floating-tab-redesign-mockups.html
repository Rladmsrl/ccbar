现状的张力

"●N 跟段染色在 36 点宽里抢着说话"

当前实现把折叠条切成 80/20:上 80% 走段染色,下 20% 给 ●N badge。问题:

  • 20% 给一个圆点 + 数字太挤,排版受困
  • overflow 段(聚合多个 session)跟独立段视觉完全一样,看不出"这一段藏了 N 个"
  • 圆点的位置反复刷新,且跟最上面的段染色色相相同时会糊在一起
需要表达的信息

五件事,一条 36×108 的窄条

  1. 当前活跃 session 总数 N
  2. 每个 session 的状态颜色(thinking/working/needs-input/...)
  3. N 超过 cap 时 — overflow 聚合的提示
  4. 是否有 needs-input(紧急)
  5. 拖拽可用(drag affordance)

VARIANT №01
Ribbon

把现在的方案打磨到位 — 100% 段染色,overflow 段加叠层条纹,数字进段内,顶部一条短横线当 drag 把手。彻底丢掉 ●N。

设计语言

纯色带,每段都是一个 session。Overflow 段叠了水平条纹(每行 1px 暗线),读起来像"多张纸叠在一起",中间叠 +N 字样。顶上一条 6% 高度的短横线 = drag handle,顺便消化 macOS 的拖拽 feedback。

取舍

  • + 改动最小,沿用既有 SwiftUI 实现
  • + 信息密度最高(每段一一对应 session)
  • − 36pt 宽里的 +3 字号必须很小(~8.5pt),无障碍折损
  • − N 总数从此不直显,要数段
VARIANT №02
Pill

不再表达"每个 session 是什么状态" — 折叠条只回答一个问题:"现在 N 个 session,主要在 ___ 状态"。大字 N + dominant-state 整条染色 + 内侧细脊条做 fleet hint。

设计语言

整条 pill 用 dominant state 渲染。N 用 Instrument Serif italic 大字 — 优雅、有性格,跟开发者 dev tool 调性匹配。左侧一根 2pt 宽的"脊条"用明暗段拟态 N 个 session(像音乐均衡器,但极简)。

取舍

  • + 远观清晰:你瞄一眼就知道 N + 主要状态
  • + needs-input 整条变红 + 脉动 — 最强烈的 alert
  • − 丢失"每个 session 单独颜色"信息(脊条只是 hint,不是精确映射)
  • − 需要新的 SwiftUI 组件,工作量大
VARIANT №03
Tag-out

把 ●N 从条里赶出去 — 让 N chip 像一个小标签贴在条的中段、向桌面方向探出来。条本身 100% 留给段染色。两个图层各得其所。

设计语言

条仍是 36pt 宽的段染色 ribbon。Chip 沿水平方向往桌面探,内有圆点 + 数字,用 Geist Mono。Chip 圆点颜色 = 最高优先级 session 状态;数字 = 真实 session 数(不被 cap 限)。Chip 自带 backdrop blur,叠在桌面上不抢戏。

取舍

  • + 两个信息层(状态色 + 数字)各享完整空间
  • − 增加 hit-test 面积(chip 可被误点)
  • − macOS NSPanel frame 要扩到 chip 之外,跟现有 36pt 假设不兼容
  • − 跟全屏 app 有可能视觉打架
VARIANT №04
Slats

每个 session 是条内一根细长发光的彩色横线 — 像一组 LED 指示灯。Overflow 表达为"末段更密的一簇线",视觉密度本身就是 count。没有数字,没有圆点。

设计语言

每根 slat 是 1.5pt 高的发光彩条,左右 22% 留白(看着像 LED 装在条内)。Overflow region 用一根 1px dashed 暗线分开,后面的 slat 更窄 + 半透明(53% 不透明)。Needs-input slat 高度 +1pt、亮度 +50%、加发光。整个条的可视语言更"硬件设备"。

取舍

  • + 美学最有性格 — 像一件设备而非 UI
  • + 数量增长到 10+ 也很优雅(更密)
  • − 没有具体数字 — 数 session 要看一会儿
  • − 跟现有 TabGlowOverlay 完全是另一套实现,改造成本最大
回到取舍 · summary

想想这条 36pt 到底是个什么?

№01 · Ribbon

"一根 信息密度 最大的彩色色带" — 沿用、打磨现有方案。改 overflow 段视觉 + 拿掉 ●N。

№02 · Pill

"一个 会变色的大数字" — 重心给 N,牺牲每条 session 的状态精度。最远观友好。

№03 · Tag-out

"条 + 旁挂的 chip" — 接受空间溢出,信息分层最干净。物理空间换了视觉清晰。

№04 · Slats

"一台微型 LED 指示器" — 美学最有性格,改造成本最高,数字感最弱。
